# Water Report Analysis MVP需求文档

## 项目背景
开发一款面向个人和企业用户的水质报告分析网站，支持多格式水质报告上传，自动解析数据并结合各国饮用水标准判断水质安全性，重点输出水质是否可饮用的结论及净化方案建议。无需用户账号，支持即时上传即得结果。

---

## 功能需求

### 1. 用户流程
- 访问网站 → 选择国家/地区标准 → 上传水质报告文件 → 系统解析数据 → 输出分析报告和净水建议 → 用户查看结果

### 2. 文件格式支持
- PDF（水质报告标准格式，含结构化文本或扫描件）
- 图片（拍照、扫描的水质报告）
- Excel（表格数据）
- Word 文档（带水质检测数据的报告）

### 3. 国家/地区选择
- 提供下拉菜单选择国家/地区
- 根据选择加载对应饮用水标准（WHO、中国GB、美国EPA等）
- 标准以JSON格式存储，方便扩展和维护

### 4. 数据解析与处理
- 支持各种文件格式的文字和表格数据提取（PDF解析、OCR识别、Excel读取、Word读取）
- 结构化抽取出关键指标（pH、重金属、浊度、细菌含量等）
- 与所选标准进行数值比对，判断是否合格、超标

### 5. AI报告解读
- 调用OpenAI GPT-3.5 API，结合检测数据和标准，生成自然语言的水质安全简明解读
- 对超标指标给出至少三条净化处理建议
- 其中第2、3条建议作为付费内容，基础版本只显示第一条
- 解读内容包括水质是否可饮用、风险提示及净化建议

### 6. 流程无账号设计
- 用户无需注册和登录，上传即用
- 不保存用户上传的原始文件和敏感信息，仅缓存解析后的结构数据用于计算
- 响应速度快，确保用户体验流畅

---

## 技术架构建议

### 1. 前端
- React+Tailwind CSS打造简洁交互页面
- 支持文件上传、国家选择、结果展示界面
- 异步请求与后端API通信

### 2. 后端
- 文件格式解析：Python服务，使用pdfplumber、Tesseract、openpyxl、python-docx等库
- 数据标准库管理：JSON格式存储饮用水国家标准阈值
- AI接口组装：调用OpenAI GPT-3.5进行报告解读
- 业务逻辑：指标比对、付费逻辑（解锁更多净化建议）

### 3. 部署
- Cloudflare Workers作为边缘计算，做请求转发与AI接口调用，减少延迟
- 文件解析由轻量云函数或服务器完成，保障性能需求
- 不存储用户上传文件，确保合规安全

---

## 付费设计

- 免费用户可获得第一条净化建议和全文AI解读
- 付费用户解锁更多净化方案和专业解读细节
- 付费功能由前端弹窗/入口引导，并通过支付系统实现
